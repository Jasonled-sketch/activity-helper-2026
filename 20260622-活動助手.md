## 3. 已做的決策（為什麼長這樣）

- **去青商化**：產品名「活動強力助手2026」，要**模組化當產品賣**。
- **schema-driven 架構**：全部欄位寫在一個 `SCHEMA` 設定物件，畫面自動長出來。
  → **白標換客戶 = 只改 `SCHEMA` + `BRAND`，不動程式邏輯。**
- **公益 / 組織 雙模式**：新增活動先選性質；公益會多顯示公益專屬欄位，組織只顯示內部欄位。
- **預算逐項可編輯**：每行「名稱 / 數量 / 備註 / 金額」一行，含快捷建議按鈕，自動加總。
- **狀態燈號 = 單一循環鈕**：點一下換下一個；儀表板宮格圓點也能直接點。
- **負責人 + 狀態鈕同一行**。
- 砍掉「預算核准」；「贊助管理」從 A3 移到 **C3 致謝感謝**。

## 4. 架構重點（給工程接手 / Claude）

**檔案**：單一 `index.html`。`<style>` + HTML 骨架 + 一大段 `<script>`（無框架、無相依）。

**資料**：`localStorage`，key = `qsc3x4_db_v2`。結構：
```
DB = { orgs:[{id,name}],
       events:[{ id, orgId, name, category, type, date, place, lead,
                 modules:{ A1:{status, owner, values:{}}, ... C4:{} } }],
       ui:{ orgId, role } }
```
> 之後要換 Firestore，介面照舊、UI 不用大改。

**SCHEMA**：12 模組（`A1`–`C4`），每個 `{phase, title, purpose, groups:[{g:'群組名', cat?, f:[欄位...]}]}`。

**欄位語法**：`"名稱|型別|選項以逗號分隔|w"`
- 型別：`text`(預設) `area` `num` `money` `date` `select` `multi` `bool` `person` `photo` `lineitems`
- 第 4 段 `w` = **公益限定欄位**（組織模式自動隱藏）
- group 加 `cat:'welfare'` = **整組公益限定**

**關鍵函式**：
- `isWelfare(e)` / `moduleFields(code, welfare)` → 依活動性質過濾要顯示的欄位
- `computeProgress(code, values, welfare)` → 只算「看得到的」欄位的完成度
- `sumLineItems(values, group)` → 預算逐項加總（儀表板「預算速覽」用）
- `cycleStatus` / `cycleCellStatus` / `nextStatus` → 狀態燈號循環
- `renderEvents` / `renderDashboard` / `renderModule` → 三個畫面
- `fieldHTML` / `saveModule` → schema 表單的渲染與存檔（含 lineitems 收集）

**多租戶**：`orgs → events → modules`（右上角可切換 / 新增組織）。

**白標入口**：檔案最上面的 `BRAND` 與 `SCHEMA` 兩塊。
